# Event System

The current event system is minimal. `SnapshotEvent` is implemented, while anomaly, status, and UI events are not yet formalized.

## Implemented Event

`cameras.camera_events.SnapshotEvent`

Fields:

- `camera_id`: string.
- `frame`: OpenCV frame object.
- `timestamp`: Unix timestamp in seconds.

Flow:

```text
CameraClient
  -> SnapshotEvent
  -> CameraManager._safe_snapshot_callback
  -> Supervisor.on_camera_snapshot
  -> processing pipeline
```

## Current Anomaly Reporting

`CycleImagePipeline._report_anomaly()` prints:

- camera ID.
- reason.
- similarity.
- anchor ID.

There is no durable anomaly event object yet.

## Suggested Separation

Future implementation should separate:

- Detection events: facts generated by pipelines.
- Transport events: websocket messages sent to UI.
- Persistence records: database schema for audit/history.
- Commands: UI or operator actions such as pause/resume/acknowledge.
